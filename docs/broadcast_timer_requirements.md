# 放送用フレーム精度・Webタイマー 要件定義書 (v1.0)

## 1. プロジェクト概要
放送業界の事前収録や進行管理において、ミリ秒単位の誤差も許されないフレーム精度の計測を行うためのWebベース・タイマーシステム。

## 2. システム構成
- **プラットフォーム**: Webブラウザ（デスクトップPC推奨）
- **運用形態**: 同一デバイス内でのマルチウィンドウ運用（操作画面と表示画面の分離）
- **画面間通信**: `BroadcastChannel API` による超低遅延同期
- **精度管理**: `performance.now()` を基準としたシステムクロック同期方式

## 3. 機能要件

### 3.1 タイマー制御ロジック
- **動作モード**: 
    - カウントアップ / カウントダウン
        - すべて手動でのモード切り替え（自動遷移なし）
        - **フレームレート（FPS）設定**: 
            - 23.976 / 24 / 25 / 29.97 (DF/NDF) / 30 / 50 / 59.94 (DF/NDF) / 60
                - ドロップフレーム（DF）時はSMPTE準拠の補正計算を適用
                - **日またぎ（ロールオーバー）処理**: 
                    - `23:59:59:ff` の次は `00:00:00:00` となる24時間循環ロジック
                        - カウントダウン終了（0到達）は、放送マスター上の「24:00」と同義として処理

                        ### 3.2 画面構成とUI仕様

                        #### A. 設定画面（コントローラー）
                        - **操作UI**: START / STOP / RESET ボタン、モード切替トグル
                        - **数値入力**: `HH:mm:ss:ff` 形式（テンキーによる連続入力に対応）
                        - **画面制御**: 表示用ウィンドウ（子画面）の起動ボタン
                        - **カラー切替**: ダークモード/ライトモードのグローバル変更

                        #### B. 表示画面（モニター）
                        - **視認性**: 数字のみを画面いっぱいに表示（等幅フォント採用）
                        - **全画面表示**: 全画面表示（Fullscreen API）への対応
                        - **保護機能**: マウスポインタの自動非表示（一定時間無操作時）

                        ### 3.3 外観・デザイン
                        - **ミニマリズム**: 装飾を完全に排し、情報の視認性を最優先
                        - **カラーパレット**: 
                            - ダークモード: 背景 `#000000` / 文字 `#FFFFFF`
                                - ライトモード: 背景 `#FFFFFF` / 文字 `#000000`

                                ## 4. 操作・ショートカット仕様

                                | キー | 機能 | 備考 |
                                | :--- | :--- | :--- |
                                | **Space** | Start / Stop | タイマーの一時停止・再開 |
                                | **Enter** | Set / Commit | 数値入力の確定 |
                                | **R** | Reset | 設定された初期値にリセット |
                                | **F** | Fullscreen | 表示画面を全画面表示にする（表示画面側で有効） |
                                | **D** | Color Mode | ダーク/ライトモードの切り替え |
                                | **Up / Down** | ±1 Frame | 1フレーム単位の微調整 |

                                ## 5. 非機能要件
                                - **高精度描画**: `requestAnimationFrame` を使用し、リフレッシュレートに合わせた描画更新
                                - **スリープ防止**: `Screen Wake Lock API` により、動作中の画面消灯を抑制
                                - **可用性**: ログイン、保存、設定の永続化をあえて行わない、即時利用可能なステートレス設計

                                ## 6. 技術実装のヒント
                                - **時間管理**: `setInterval` は使用せず、開始時と現在時の差分ミリ秒からFPSに応じた累積フレーム数を計算すること。
                                - **同期**: 設定画面側を「Master（親）」、表示画面側を「Slave（子）」とし、開始基準時刻（Reference Time）を `BroadcastChannel` で共有すること。
